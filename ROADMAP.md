# AnimeHunt — ROADMAP

> Шкала готовности: **1% → 100%** (100% = задеплоен на хостинг и готов к реальным пользователям)
>
> Приоритеты внутри каждого блока:
> - 🔴 **Критическое** — без этого блок нельзя считать завершённым
> - 🟠 **Важное** — сильно влияет на качество / безопасность
> - 🟡 **Нужное** — заметно улучшает опыт
> - 🔵 **Потом** — хорошо иметь, но не блокирует

---

## 📍 Текущий статус проекта: ~42%

Проект имеет полноценное ядро: аутентификация, каталог аниме и персонажей с фильтрами, страницы аниме и персонажей, система оценок/рецензий/комментариев с форматированием, медиа-галерея с модерацией, шипперинг, избранное, профили пользователей с приватностью, admin-панель. Работает локально на `localhost`. До продакшена не хватает: security-хардения, работы с переменными окружения, оптимизации, SEO, CI/CD и непосредственно деплоя.

## Этап 3 — Качество и безопасность (50% → 70%) 🔄 В ПРОЦЕССЕ

### 🔴 Критическое
- [ ] **Вынести все секреты в `.env`** — сейчас `JWT_SECRET=supersecretkey123` и `DB_PASSWORD=postgres` (менять перед деплоем обязательно)
- [ ] **Убрать хардкод `localhost:3001`** из `src/api/api.js` и `src/pages/Profile.jsx` → использовать `VITE_API_URL` из `.env`
- [ ] **CORS**: заменить `origin: "http://localhost:5173"` на переменную окружения
- [ ] **Helmet.js** — HTTP security headers (X-Frame-Options, CSP, HSTS и др.)
- [ ] **Rate limiting** — `express-rate-limit` на `/api/auth/*` (защита от брутфорса)
- [ ] **Валидация входных данных** на сервере — добавить `joi` или `zod` для всех POST/PUT роутов
- [ ] **Создать `.env.example`** — шаблон с описанием всех переменных (без реальных значений)
- [ ] **Сжатие ответов** — `compression` middleware

### 🟠 Важное
- [ ] **Пагинация в каталоге** — сейчас `limit=100` с offset, нужны кнопки страниц или бесконечная прокрутка
- [ ] **SQL-инъекции** — уже используется параметризация, но нужен audit всех запросов (особенно динамические WHERE)
- [ ] **Размер загружаемых изображений** — сжатие через `sharp` перед сохранением (сейчас base64 до 20 МБ идёт в JSON и в БД)
- [ ] **Хранилище изображений** — локальная папка `server/uploads/` не масштабируется, пока не трогаем;
- [ ] **Refresh token** — сейчас JWT живёт 30 дней без возможности отзыва; добавить refresh + blacklist
- [ ] **Логирование** — `morgan` для HTTP-логов, структурированные логи ошибок

### 🟡 Нужное
- [ ] **Обработка ошибок** — глобальный error handler в Express; `Error Boundary` в React
- [ ] **Загрузка состояний** — скелетоны вместо спиннеров в каталоге и на главной
- [ ] **SEO**: заменить `<title>site</title>` на динамические мета-теги (название аниме/персонажа), Open Graph
- [ ] **Смена пароля** — форма в настройках профиля
- [ ] **Удаление аккаунта** — возможность для пользователя
- [ ] **Модерация комментариев** — мод/админ может удалять чужие комментарии
- [ ] **Добавить списки отложено и дропнуто** - сейчас у пользователя только 3 варианта списков(смотрю, просмотрено, запланировано), нужно еще 2 варианта
- [ ] **Редактирование комментария** — в течение N минут после публикации

### 🔵 Потом
- [ ] **Email-верификация** при регистрации (через nodemailer / Resend)
- [ ] **Восстановление пароля** по email
- [ ] **Уведомления** (ответ на комментарий, лайк рецензии) — на первом этапе просто badge в шапке
